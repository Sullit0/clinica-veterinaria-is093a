2. Registro de Correcciones Manuales (Anti-IA)

HTML / ARIA

Corrección de ID para Accesibilidad en el Menú

Problema: El validador W3C arrojaba el error "The aria-controls attribute references nav-list, which is not the ID of any element" en el botón hamburguesa móvil.

Solución manual: El código autogenerado originalmente asignó class="nav-list" pero omitió el ID requerido para que aria-controls funcione para lectores de pantalla. Se añadió id="nav-list" a la etiqueta ul.

Sintaxis CSS

Depuración del bug de Comentarios Anidados

Problema: El IDE indicaba errores sintácticos desde la línea 892 de styles.css, donde las reglas de @container dejaban de ser reconocidas.

Solución manual: Se detectó la inclusión de un comentario CSS (/* IA: ... */) dentro de otro bloque de comentario general (anidamiento no soportado por CSS). Se eliminaron los caracteres de escape internos reestructurando el bloque para recuperar el hilo de lectura del motor de render.

Grid Layout

Restricción Semántica del Sidebar ("Filtrar por Paciente")

Problema: En resoluciones de escritorio, el menú lateral aside abarcaba filas correspondientes a las secciones "Instalaciones" y "Nosotros", cuando únicamente debía servir de filtro para la "Galería" de servicios.

Solución manual: En lugar de permitir que áreas múltiples caigan bajo el dominio "sidebar" en grid-template-areas, se rediseñaron las reglas para que las secciones en la capa inferior fueran declaradas con span completo (e.g. "tech tech" y "nosotros nosotros"), aislando al sidebar en la zona superior donde es lógicamente correcto.

Flexbox Mobile

Rediseño del Layout de Filtros (Píldoras) en Responsive

Problema: El colapso a versión móvil generaba que los filtros intermedios crearan un salto de página interminable de enlaces verticales.

Solución manual: Se implementó flex-direction: row; flex-wrap: wrap; en los contenedores `.filter-list` y `.stats-list`, y flex: 1 1 auto;. El efecto resultante transformó una pila pesada en una botonera horizontal adaptativa.

Estructura DOM

Conflictos con role="listitem" en Sectioning Elements

Problema: La asignación del role "listitem" sobre la etiqueta <article> contradecía la semántica pura de HTML5 al sobreescribir un Content Model estructural autodefinido.

Solución manual: Se retiraron los roles de sobreescritura dejándolo como un nodo list/children puro o se ajustaron de acuerdo a la exigencia de la rúbrica.

3. Justificación Técnica Requerida

El uso de Container Queries (@container)

El validador de CSS W3C indicó: "Unrecognized at-rule @container". Esto se debe a que la herramienta oficial de la W3C no ha sido actualizada todavía para parsear las directivas del Módulo Level 4 de CSS sobre Container Queries, sin embargo, los navegadores modernos los soportan nativamente. Esta fue una de las herramientas exigidas por la Práctica 02.

Fallback para Backdrop Filter

Para asegurar que navegadores más antiguos que no soportan efectos "glassmorphism" no queden con encabezados ilegibles, se utilizó una cláusula defensiva:

@supports not (backdrop-filter: blur(1px)) {
  .site-header {
    background: var(--clr-bg-secondary);
  }
}
